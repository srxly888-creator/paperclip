## Paperclip 是什么？

# 零人公司的开源编排

**如果 OpenClaw 是_雇员_，Paperclip 就是_公司_**

Paperclip 是 Node.js 服务器和 React UI，用于协调 AI 智能体团队来运行业务。自带智能体、分配目标并通过一个控制台跟踪智能体的工作和成本。

它看起来像一个任务管理器，但在幕后它有组织架构图、预算、治理、目标调整和智能体协调。

**管理业务目标，而不是拉取请求。**

|        |步骤|示例|
| ------ | --------------- | ------------------------------------------------------------------ |
| **01** |定义目标 | _“以 100 万美元的 MRR 打造排名第一的人工智能笔记应用程序。”_ |
| **02** |聘请团队|首席执行官、首席技术官、工程师、设计师、营销人员——任何机器人、任何提供商。 |
| **03** |批准并运行 |审查策略。设定预算。打吧。从控制台进行监控。  |

<br/>

> **即将推出：Clipmart** — 一键下载并运行整个公司。浏览预构建的公司模板 - 完整的组织结构、智能体配置和技能 - 并在几秒钟内将它们导入到您的 Paperclip 实例中。

## Paperclip 适合您，如果

- ✅ 你想建立**自主人工智能公司**
- ✅ 您**协调许多不同的智能体**（OpenClaw、Codex、Claude、Cursor）以实现共同目标
- ✅ 您有**20 个同时打开的 Claude Code 终端**并且无法追踪每个人在做什么
- ✅ 您希望智能体**24/7** 自主运行，但仍希望审核工作并在需要时介入
- ✅ 您想要**监控成本**并执行预算
- ✅ 您想要一个**感觉像使用任务管理器**的智能体管理流程
- ✅ 您想要**通过手机**管理您的自主业务**

## 为什么 Paperclip 很特别

Paperclip 正确处理硬编排细节。|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子执行。** |任务结帐和预算执行是原子的，因此不会重复工作，也不会出现失控的支出。                      |
| **持久智能体状态。** |智能体通过心跳恢复相同的任务上下文，而不是从头开始。                     |
| **运行时技能注入。** |智能体可以在运行时学习 Paperclip 工作流程和项目上下文，而无需重新培训。                      |
| **回滚治理。** |强制执行批准门，修改配置更改，并且可以安全地回滚不良更改。        |
| **目标意识执行。** |任务具有完整的目标血统，因此智能体始终可以看到“原因”，而不仅仅是标题。                        |
| **便携式公司模板。** |具有秘密清理和碰撞处理功能的导出/导入组织、智能体和技能。                          |
| **真正的多公司隔离。** |每个实体都是公司范围的，因此一次部署可以运行许多具有单独数据和审计跟踪的公司。 |

## Paperclip 不是什么

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天机器人。** |客服人员有工作，没有聊天窗口。                                                                                  |
| **不是智能体框架。** |我们不会告诉您如何建立智能体。我们告诉您如何经营一家由他们组成的公司。                                |
| **不是工作流程构建器。** |没有拖放管道。 Paperclip 通过组织架构图、目标、预算和治理为公司建模。            |
| **不是提示管理器。** |智能体带来自己的提示、模型和运行时。 Paperclip 管理他们工作的组织。
| **不是单智能体工具。** |这是针对团队的。如果您有一名智能体，则可能不需要 Paperclip。如果你有二十个——你肯定有。 |
| **不是代码审查工具。** | Paperclip 协调工作，而不是拉取请求。带来您自己的审核流程。                                       |

## 快速入门

开源。自托管。无需 Paperclip 帐户。

```bash
npx paperclipai onboard --yes
```

或者手动：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这将启动位于 `http://localhost:3100` 的 API 服务器。嵌入式 PostgreSQL 数据库会自动创建 - 无需设置。

> **要求：** Node.js 20+、pnpm 9.15+

## 常见问题解答

**典型的设置是什么样的？**
在本地，单个 Node.js 进程管理嵌入式 Postgres 和本地文件存储。对于生产环境，将其指向您自己的 Postgres 并根据您的喜好进行部署。配置项目、智能体和目标——智能体负责剩下的事情。如果您是个体企业家，您可以使用 Tailscale 随时随地访问 Paperclip。然后您可以部署到例如Vercel 当你需要的时候。

**我可以经营多家公司吗？**
是的。一次部署可以运行无限数量的公司，并且数据完全隔离。

**Paperclip 与 OpenClaw 或 Claude Code 等智能体有何不同？**
Paperclip_使用_那些智能体。它将他们编排成一家公司——有组织架构图、预算、目标、治理和责任。

**为什么我应该使用 Paperclip 而不是仅仅将我的 OpenClaw 指向 Asana 或 Trello？**
智能体编排在如何协调谁签出工作、如何维护会话、监控成本、建立治理方面具有微妙之处 - Paperclip 可以为您做到这一点。

（路线图上有自带门票系统）

**智能体是否持续运行？**
默认情况下，智能体根据计划的检测信号和基于事件的触发器（任务分配、@-提及）运行。您还可以挂接连续智能体，例如 OpenClaw。你带上你的智能体人和Paperclip坐标。

## 发展

```bash
pnpm dev              # Full dev (API + UI, watch mode)
pnpm dev:once         # Full dev without file watching
pnpm dev:server       # Server only
pnpm build            # Build all
pnpm typecheck        # Type checking
pnpm test:run         # Run tests
pnpm db:generate      # Generate DB migration
pnpm db:migrate       # Apply migrations
```

完整的开发指南请参见 [doc/DEVELOPING.md](doc/zh-CN/DEVELOPING.md)。

# v0.3.0

> 发布时间：2026-03-09

## 亮点

- **新适配器：Cursor、OpenCode 和 Pi** — Paperclip 现在支持三个额外的本地编码智能体。 Cursor 和 OpenCode 作为一流的适配器集成，具有模型发现、运行日志流和技能注入功能。 Pi 添加了具有成本跟踪功能的本地 RPC 模式。所有三个都与 Claude Code 和 Codex 一起出现在入门向导中。 ([#62](https://github.com/paperclipai/paperclip/pull/62)、[#141](https://github.com/paperclipai/paperclip/pull/141)、[#240](https://github.com/paperclipai/paperclip/pull/240)、[#183](https://github.com/paperclipai/paperclip/pull/183)、@aaaaron、@Konan69、@richardanaya)
- **OpenClaw 网关适配器** — 新的仅网关 OpenClaw 流取代了旧适配器。它使用严格的 SSE 流，支持设备密钥配对，并通过加入令牌验证处理基于邀请的加入。 ([#270](https://github.com/paperclipai/paperclip/pull/270))
- **收件箱和未读语义** - 问题现在跟踪每个用户的阅读状态。未读指示器出现在收件箱、控制台和浏览器选项卡（蓝点）中。收件箱徽章包括加入请求和批准，收件箱排序以警报为重点。 （[#196]（https://github.com/paperclipai/paperclip/pull/196），@hougangdev）
- **PWA 支持** — UI 作为可安装的渐进式 Web 应用程序提供，带有服务工作人员和增强的清单。 Service Worker 使用网络优先策略来防止内容过时。
- **智能体创建向导** — 新的选择模式和全页配置流程使添加智能体变得更加容易。侧边栏 AGENTS 标题现在有一个快速添加按钮。

## 改进- **Markdown 中的美人鱼图** — 围栏 `mermaid` 块在问题评论和描述中呈现为图表。
- **实时运行输出** — 通过 WebSocket 实时运行详细信息页面流输出，并合并增量和重复数据删除的提要项目。
- **将评论复制为 Markdown** — 每个评论标题都有一个一键复制为 Markdown 按钮。
- **重试失败的运行** — 失败和超时的运行现在会在运行详细信息页面上显示“重试”按钮。
- **可单击项目状态** — 项目属性窗格中的状态芯片现在可单击以进行快速更新。
- **滚动到底部按钮** — 向上滚动时，问题详细信息和运行页面会显示浮动滚动到底部按钮。
- **数据库备份 CLI** — `paperclipai db:backup` 允许您按需快照数据库，并可选择自动调度。
- **禁用注册** — 新的 `auth.disableSignUp` 配置选项（和 `AUTH_DISABLE_SIGNUP` 环境变量）允许操作员锁定注册。 （[#279]（https://github.com/paperclipai/paperclip/pull/279），@JasonOA888）
- **删除重复的短名称** — 智能体和项目短名称现在在创建和更新时自动删除重复项，而不是拒绝重复项。 （[#264]（https://github.com/paperclipai/paperclip/pull/264），@mvanhorn）
- **人类可读的角色标签** — 智能体列表和属性窗格显示友好的角色名称。 （[#263]（https://github.com/paperclipai/paperclip/pull/263），@mvanhorn）
- **受让人选择器排序** — 首先显示最近的选择，然后按字母顺序显示。
- **移动布局优化** — 跨问题、收件箱和控制台统一 GitHub 样式的问题行。改进了移动设备上的弹出窗口滚动、命令面板居中和属性切换。 （[#118]（https://github.com/paperclipai/paperclip/pull/118），@MumuTW）
- **邀请用户体验改进** — 邀请链接自动复制到剪贴板、设置中仅片段流程、10 分钟邀请 TTL 以及更清晰的网络主机指导。
- **评论上的永久链接锚点** — 每个评论都有一个稳定的锚链接和一个 GET-by-ID API 端点。
## 改进

- **Markdown 中的美人鱼图** — 围栏 `mermaid` 块在问题评论和描述中呈现为图表。
- **更新了模型列表** — 将 `claude-sonnet-4-6`、`claude-haiku-4-6` 和 `gpt-5.4` 添加到适配器模型常量中。 （[#293]（https://github.com/paperclipai/paperclip/pull/293），[#110]（https://github.com/paperclipai/paperclip/pull/110），@cpfarhood，@artokun）
- **Playwright e2e 测试** — 新的端到端测试套件涵盖了入门向导流程。

## 修复- **运行日志中的秘密编辑** — 来自秘密的环境变量现在按来源进行编辑，并具有一致的 `secretKeys` 跟踪。 （[#261]（https://github.com/paperclipai/paperclip/pull/261），@mvanhorn）
- **SPA catch-all 500s** — 服务器在 catch-all 路由中提供缓存的 `index.html`，并在 `sendFile` 中使用 `root`，防止点文件路径和 SPA 刷新上出现 500 错误。 （[#269]（https://github.com/paperclipai/paperclip/pull/269），[#78]（https://github.com/paperclipai/paperclip/pull/78），@mvanhorn，@MumuTW）
- **不匹配的 API 路由返回 404 JSON** — 之前落入 SPA 处理程序。 （[#269]（https://github.com/paperclipai/paperclip/pull/269），@mvanhorn）
- **智能体唤醒逻辑** — 当问题移出积压工作时，智能体会唤醒，跳过自己评论的自我唤醒，并跳过积压状态更改的唤醒。待批准的智能体不包括在心跳计时器中。 ([#159](https://github.com/paperclipai/paperclip/pull/159)、[#154](https://github.com/paperclipai/paperclip/pull/154)、[#267](https://github.com/paperclipai/paperclip/pull/267)、[#72](https://github.com/paperclipai/paperclip/pull/72)、@Logesh-waran2003、@cschneid、@mvanhorn、@STRML)
- **运行日志 fd 泄漏** — 修复了日志附加中导致 `spawn EBADF` 错误的文件描述符泄漏。 （[#266]（https://github.com/paperclipai/paperclip/pull/266），@mvanhorn）
- **500 错误日志记录** — 错误日志现在包括实际的错误消息和请求上下文，而不是通用的 pino-http 输出。
- **布尔环境解析** - `parseBooleanFromEnv` 不再默默地将常见的 true 值视为 false。 （[#91]（https://github.com/paperclipai/paperclip/pull/91），@zvictor）
- **载入环境默认值** - `onboard` 现在可以正确地从环境变量中导出机密，并报告 `local_trusted` 模式下忽略的暴露设置。 （[#91]（https://github.com/paperclipai/paperclip/pull/91），@zvictor）
- **Windows 路径兼容性** — 迁移路径使用 `fileURLToPath` 进行 Windows 安全解析。 （[#265]（https://github.com/paperclipai/paperclip/pull/265），[#413]（https://github.com/paperclipai/paperclip/pull/413），@mvanhorn，@online5880）
- **HTTP 上的安全 cookie** — 禁用普通 HTTP 部署的安全 cookie 标志，以防止身份验证失败。 （[#376]（https://github.com/paperclipai/paperclip/pull/376），@dalestubblefield）
- **URL 编码** — `buildUrl` 拆分路径和查询以防止 `%3F` 编码问题。 （[#260]（https://github.com/paperclipai/paperclip/pull/260），@mvanhorn）
- **验证可信来源** — 有效的可信来源和允许的主机名现在可以在公共模式下正确应用。 （[#99]（https://github.com/paperclipai/paperclip/pull/99），@zvictor）
- **UI 稳定性** — 修复了提示模板清空时的空白屏幕、搜索 URL 同步导致重新渲染、收件箱中问题标题溢出以及侧边栏徽章计数（包括批准）。 ([#262](https://github.com/paperclipai/paperclip/pull/262)、[#196](https://github.com/paperclipai/paperclip/pull/196)、[#423](https://github.com/paperclipai/paperclip/pull/423)、@mvanhorn、@hougangdev、@RememberV)

# 智能体运行时指南

状态：面向用户的指南
最后更新: 2026-02-17
受众：在 Paperclip 中设置和运行智能体的运营商

## 1. 这个系统的作用

Paperclip 中的智能体不会连续运行。
它们以**心跳**的方式运行：由唤醒触发的短执行窗口。

每一次心跳：

1. 启动配置的智能体适配器（例如，Claude CLI 或 Codex CLI）
2. 给出当前提示/上下文
3. 让它一直工作，直到退出、超时或被取消
4. 存储结果（状态、令牌使用情况、错误、日志）
5. 实时更新UI

## 2. 当智能体醒来时

可以通过四种方式唤醒智能体：

- `timer`：预定间隔（例如每5分钟）
- `assignment`：当工作被分配/签出给该智能体时
- `on_demand`：手动唤醒（按钮/API）
- `automation`：系统触发的唤醒以实现未来的自动化

如果智能体已在运行，则会合并新的唤醒，而不是启动重复的运行。

## 3. 每个智能体要配置什么

## 3.1 适配器选择

常见选择：

- `claude_local`：运行本地`claude` CLI
- `codex_local`：运行本地`codex` CLI
- `process`：通用 shell 命令适配器
- `http`：调用外部HTTP端点

对于 `claude_local` 和 `codex_local`，Paperclip 假定 CLI 已在主机上安装并经过身份验证。

## 3.2 运行时行为

在智能体运行时设置中，配置心跳策略：

- `enabled`：允许预定的心跳
- `intervalSec`：计时器间隔（0 = 禁用）
- `wakeOnAssignment`：分配工作时唤醒
- `wakeOnOnDemand`：允许 ping 式按需唤醒
- `wakeOnAutomation`：允许系统自动唤醒

## 3.3 工作目录和执行限制

对于本地适配器，设置：

- `cwd`（工作目录）
- `timeoutSec`（每个心跳的最大运行时间）
- `graceSec`（超时/取消后强行杀死前的时间）
- 可选的环境变量和额外的 CLI 参数
- 在保存之前在智能体配置中使用**测试环境**来运行适配器特定的诊断

## 3.4 提示模板

您可以设置：

- `promptTemplate`：用于每次运行（首次运行和恢复会话）

模板支持 `{{agent.id}}`、`{{agent.name}}` 等变量和运行上下文值。

## 4. 会话恢复行为

Paperclip 存储可恢复适配器的会话 ID。

- 下一个心跳会自动重用已保存的会话。
- 这提供了心跳的连续性。
- 如果上下文变得陈旧或混乱，您可以重置会话。

在以下情况下使用会话重置：

- 你显着改变了提示策略
- 智能体陷入了恶性循环
- 你想要一个干净的重启

## 5. 日志、状态和运行历史记录对于每次心跳运行，您都会得到：

- 运行状态（`queued`、`running`、`succeeded`、`failed`、`timed_out`、`cancelled`）
- 错误文本和 stderr/stdout 摘录
- 适配器可用时的令牌使用/成本
- 完整日志（存储在核心运行行之外，针对大输出进行了优化）

在本地/开发设置中，完整日志存储在磁盘上配置的运行日志路径下。

## 7. 常见操作模式

## 7.1 简单的自治循环

1. 启用定时器唤醒（例如每300秒）
2. 保持分配唤醒开启
3. 使用有重点的提示模板
4. 观察运行日志并随时间调整提示/配置

## 7.2 事件驱动循环（较少持续轮询）

1. 禁用定时器或设置较长的间隔
2. 保持分配唤醒启用
3. 使用按需唤醒进行手动微移

## 7.3 安全第一循环

1、超时时间短
2. 保守提示
3. 监控错误并在需要时快速取消
4. 出现漂移时重置会话

## 8. 故障排除

如果运行多次失败：

1. 检查适配器命令可用性（`claude`/`codex` 安装并登录）。
2. 验证 `cwd` 是否存在且可访问。
3. 检查运行错误 + stderr 摘录，然后检查完​​整日志。
4. 确认超时时间是否太低。
5. 重置会话并重试。
6. 如果智能体导致重复出现错误更新，请暂停智能体。

典型故障原因：

- CLI 未安装/验证
- 错误的工作目录
- 适配器参数/env 格式错误
- 提示过于宽泛或缺少约束
- 处理超时

Claude具体说明：

- 如果在适配器环境或主机环境中设置了 `ANTHROPIC_API_KEY`，则 Claude 使用 API-key 身份验证而不是订阅登录。 Paperclip 在环境测试中将其视为警告，而不是硬错误。

## 9. 安全和风险说明

本地 CLI 适配器在主机上以非沙盒方式运行。

这意味着：

- 提示指示很重要
- 配置的凭据/环境变量是敏感的
- 工作目录权限很重要

尽可能从最低权限开始，并避免在广泛的可重用提示中暴露秘密，除非故意需要。

## 10. 最小设置清单

1. 选择适配器（`claude_local` 或 `codex_local`）。
2. 将`cwd` 设置为目标工作空间。
3. 添加bootstrap+普通提示模板。
4. 配置心跳策略（计时器和/或分配唤醒）。
5. 触发手动唤醒。
6. 确认运行成功并记录会话/令牌使用情况。
7. 观看实时更新并迭代提示/配置。

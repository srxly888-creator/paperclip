# 贡献指南

[English](./CONTRIBUTING.md) | **简体中文**

感谢您愿意贡献！

我们非常感谢小修复和深思熟虑的大改变。

## 让 Pull 请求被接受的两种途径

### 路径 1：小而集中的更改（最快的合并方式）
- 选择**一个**明确的事情来修复/改进
- 修改**尽可能少的文件数量**
- 确保变更非常有针对性且易于审核
- 所有自动检查均通过（包括 Greptile 注释）
- 没有新的 lint/测试失败

当它们干净时，它们几乎总是会很快合并。

### 路径 2：更大或有影响力的改变
- **首先**在 Discord → #dev 频道中谈论它
  → 描述您想要解决的问题
  → 分享粗略的想法/方法
- 一旦达成粗略协议，就开始构建它
- 在您的PR中包括：
  - 变更前/后的屏幕截图（如果用户界面/行为发生变化，则为短视频）
  - 清晰描述内容和原因
  - 证明它有效（手动测试说明）
  - 所有测试均通过
  - 所有 Greptile + 其他PR评论均得到解决

遵循这条路径的 PR 被接受的可能性**大**，即使它们很大。

## 一般规则（两条路径）
- 编写清晰的提交消息
- 保持PR标题+描述有意义
- 一项 PR = 一项逻辑变更（除非是一个小的相关群体）
- 首先在本地运行测试
- 在讨论中保持友善😄

有问题吗？只需在#dev 中提问即可——我们很乐意提供帮助。

快乐黑客！
